## コマンド

### `purus build` / `purus compile`

Purusソースファイルをコンパイルします。

```sh
# 単一ファイルのコンパイル
purus build hello.purus

# ディレクトリ内の全ファイルをコンパイル
purus build --directory src

# config.purusの設定を使用してコンパイル
purus build

# 出力先ディレクトリを指定
purus build --output build

# ヘッダーコメントなしでコンパイル
purus build --no-header hello.purus
```

| 拡張子 | 出力 |
|---|---|
| `.purus` | `.js` |
| `.cpurus` | `.cjs` |
| `.mpurus` | `.mjs` |

#### オプション

| オプション | エイリアス | 説明 |
|---|---|---|
| `--directory <dir>` | `-d` | ディレクトリ内の全 `.purus` ファイルをコンパイル |
| `--output <dir>` | `-o` | 出力先ディレクトリを指定（config.purusより優先） |
| `--no-header` | | `// Generated by Purus` ヘッダーを省略 |
| `--stdout` | | コンパイル結果を標準出力に出力 |

### `purus run`

ファイルを生成せずにPurusファイルをコンパイルして実行します。

```sh
# 単一ファイルの実行
purus run hello.purus

# ディレクトリ内の全ファイルを実行
purus run --directory src

# config.purusの設定を使用して実行
purus run
```

#### オプション

| オプション | エイリアス | 説明 |
|---|---|---|
| `--directory <dir>` | `-d` | ディレクトリ内の全 `.purus` ファイルを実行 |

### `purus check`

コンパイルせずに構文チェックのみ行います。

```sh
purus check hello.purus
```

### `purus new` / `purus create`

新しいPurusプロジェクトを対話形式で作成します。対象ディレクトリは存在しないか、空である必要があります。

```sh
# 対話形式 — プロジェクト名、npm init、依存関係を問う
purus new

# プロジェクト名を指定
purus new my-project

# すべてのプロンプトをスキップ（自動でyes）
purus new my-project -y
```

このコマンドは以下を行います：

1. プロジェクトディレクトリを作成（`src/main.purus`、`config.purus`、`.prettierrc`、`README.md`、`.gitignore`）
2. `npm init` を実行（`-y` フラグ時は `npm init -y`）
3. `package.json` にスクリプトを追加：
   - `purus` — purus CLIの実行
   - `build` — `purus build` でコンパイル
   - `compile` — `purus compile` でコンパイル
   - `exec` — `purus run` で実行
   - `format` — `prettier --write ./src` でフォーマット
   - `lint` — `purus-lint` でリント
4. オプションでdevDependenciesをインストール：`purus`、`@puruslang/linter`、`@puruslang/prettier-plugin-purus`、`prettier`

### `purus init`

現在のディレクトリでPurusプロジェクトを初期化します。`src/` と `src/main.purus` が存在しなければ作成します。

```sh
purus init
```

### `purus version`

```sh
purus version
```

### `purus help`

```sh
purus help
```

## config.purus

プロジェクトルートに `config.purus` を配置して、ビルドとリンターの設定を行います。

```
-- Purus Configuration

const entry be ///src///
const output be ///dist///
const header be true

-- Linter settings
const lint.no-var be ///warn///
const lint.indent-size be 2
const lint.max-line-length be ///off///
```

### ビルド設定

| キー | 型 | デフォルト | 説明 |
|---|---|---|---|
| `entry` | string | `"src"` | ソースディレクトリまたはファイル |
| `output` | string | `"dist"` | 出力先ディレクトリ |
| `header` | boolean | `true` | `// Generated by Purus` コメントを含める |

### リンター設定

| キー | 型 | デフォルト | 説明 |
|---|---|---|---|
| `lint.no-var` | string | `"warn"` | `var` 使用時の警告またはエラー |
| `lint.indent-size` | number | `2` | 期待されるインデントサイズ |
| `lint.max-line-length` | string/number | `"off"` | 最大行長（`"off"` で無効化） |

`purus build`、`purus run`、`purus-lint` はファイル引数がない場合、自動的に `config.purus` を読み込みます。`--output` などのコマンドラインオプションは設定ファイルの値を上書きします。
