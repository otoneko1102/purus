## Commands

### `purus build` / `purus compile`

Compile Purus source files to JavaScript.

```sh
# Compile a single file
purus build hello.purus

# Compile all files in a directory
purus build --directory src

# Compile using config.purus settings
purus build

# Specify output directory
purus build --output build

# Compile without header comment
purus build --no-header hello.purus
```

| Extension | Output |
|---|---|
| `.purus` | `.js` |
| `.cpurus` | `.cjs` |
| `.mpurus` | `.mjs` |

#### Options

| Option | Alias | Description |
|---|---|---|
| `--directory <dir>` | `-d` | Compile all `.purus` files in the directory |
| `--output <dir>` | `-o` | Specify output directory (overrides config.purus) |
| `--no-header` | | Omit the `// Generated by Purus` header |
| `--stdout` | | Print compiled output to stdout |

### `purus run`

Compile and run Purus files without generating output files.

```sh
# Run a single file
purus run hello.purus

# Run all files in a directory
purus run --directory src

# Run using config.purus settings
purus run
```

#### Options

| Option | Alias | Description |
|---|---|---|
| `--directory <dir>` | `-d` | Run all `.purus` files in the directory |

### `purus check`

Check syntax without compiling.

```sh
purus check hello.purus
```

### `purus new` / `purus create`

Create a new Purus project interactively. The target directory must not exist or must be empty.

```sh
# Interactive — prompts for project name, npm init, and dependencies
purus new

# Specify project name
purus new my-project

# Skip all prompts (auto yes)
purus new my-project -y
```

This command will:

1. Create the project directory with `src/main.purus`, `config.purus`, `.prettierrc`, `README.md`, and `.gitignore`
2. Run `npm init` (or `npm init -y` with `-y` flag)
3. Add scripts to `package.json`:
   - `purus` — run purus CLI
   - `build` — compile with `purus build`
   - `compile` — compile with `purus compile`
   - `exec` — run with `purus run`
   - `format` — format with `prettier --write ./src`
   - `lint` — lint with `purus-lint`
4. Optionally install devDependencies: `purus`, `@puruslang/linter`, `@puruslang/prettier-plugin-purus`, `prettier`

### `purus init`

Initialize a Purus project in the current directory. Creates `src/` and `src/main.purus` if they don't exist.

```sh
purus init
```

### `purus version`

```sh
purus version
```

### `purus help`

```sh
purus help
```

## config.purus

Place a `config.purus` file in your project root to configure build and lint settings.

```
-- Purus Configuration

const entry be ///src///
const output be ///dist///
const header be true

-- Linter settings
const lint.no-var be ///warn///
const lint.indent-size be 2
const lint.max-line-length be ///off///
```

### Build settings

| Key | Type | Default | Description |
|---|---|---|---|
| `entry` | string | `"src"` | Source directory or file |
| `output` | string | `"dist"` | Output directory |
| `header` | boolean | `true` | Include `// Generated by Purus` comment |

### Linter settings

| Key | Type | Default | Description |
|---|---|---|---|
| `lint.no-var` | string | `"warn"` | Warn or error on `var` usage |
| `lint.indent-size` | number | `2` | Expected indentation size |
| `lint.max-line-length` | string/number | `"off"` | Max line length (`"off"` to disable) |

`purus build`, `purus run`, and `purus-lint` read `config.purus` automatically when no file arguments are given. Command-line options like `--output` override config values.
